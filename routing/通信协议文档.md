# NS-3 与 Windows 端通信协议文档

## 1. 概述

本文档详细说明 NS-3 仿真环境与 Windows 端服务器之间的数据通信逻辑，包括数据上传流程、JSON 数据结构以及每个字段的数据获取方式。

**通信方式**: TCP Socket + HTTP POST 请求  
**数据格式**: JSON  
**API 端点**: `/api/simulation/callback`  
**上传模式**: 周期性上传（由 `m_uploadInterval` 控制间隔时间）

---

## 2. 整体通信流程

### 2.1 初始化阶段
1. **InitSocket_Comm(rp)** - 初始化 Socket 连接
   - 调用 `CreateTcpConnection_Comm` 建立 TCP 连接
   - 连接目标：`rp->m_serverIp:rp->m_serverPort`

### 2.2 数据上传循环
`UploadLoop_Comm(rp)` 函数负责周期性上传节点数据：

```
开始上传循环
  ├─ 遍历所有节点 (0 到 rp->size-1)
  ├─ 过滤：仅上传身份非 0 的节点 (IsActiveNode 检查)
  ├─ 为每个活跃节点构建 NodeReport (BuildNodeReport)
  ├─ 序列化为 JSON (SerializeNodeReportToJson_Comm)
  ├─ 发送 HTTP POST 请求 (SendMessageAndAndAndListen_Comm)
  ├─ 同步等待 Windows 端响应
  └─ 调度下一次上传 (Simulator::Schedule)
```

### 2.3 HTTP 请求格式
```http
POST /api/simulation/callback HTTP/1.1
Host: <服务器IP>:<端口>
Content-Type: application/json
Content-Length: <JSON字节数>
Connection: close

<JSON数据体>
```

## 3. JSON 数据结构详解

每次上传**单个节点**的完整 JSON 结构如下：

```json
{
  "task_id": "xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx",
  "status": "running",
  "simulation_time": 10.50,
  "node_id": 5,
  "node_identity": 2,
  "nodes": [
    {
      "scene_info": {
        "m_info": { /* 节点自身场景信息 */ },
        "neighbor_info": { /* 邻居节点统计信息 */ }
      },
      "para_info": { /* 协议参数信息 */ },
      "result_info": { /* 性能结果信息 */ }
    }
  ]
}
```

---

## 4. 顶层字段数据来源

### 4.1 task_id (任务标识)
- **类型**: `string` (UUID v4 格式)
- **来源**: `rp->m_taskId`
- **生成方式**: 通过 `GenerateUUID()` 函数生成
  - 使用随机数生成器 (`std::random_device`, `std::mt19937`)
  - 格式: `xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx`
  - 在仿真初始化时生成一次，后续所有上传使用同一个 task_id

### 4.2 status (仿真状态)
- **类型**: `string`
- **固定值**: `"running"`
- **说明**: 表示仿真正在运行中

### 4.3 simulation_time (仿真时间)
- **类型**: `double` (秒)
- **来源**: `Simulator::Now().GetSeconds()`
- **说明**: NS-3 仿真器的当前仿真时间（非真实时间）

### 4.4 node_id (节点ID)
- **类型**: `uint32_t`
- **来源**: 节点在 `rp->nodes` 容器中的索引 (0 到 `rp->size-1`)
- **说明**: 唯一标识一个仿真节点

### 4.5 node_identity (节点身份)
- **类型**: `uint8_t`
- **来源**: `ipsRouting->GetNodeIdentity()`
- **获取路径**:
  ```cpp
  node->GetObject<Ipv4>()
      ->GetRoutingProtocol()
      ->GetNodeIdentity()  // IPS 协议特有方法
  ```
- **过滤规则**: 仅上传 `node_identity != 0` 的节点

## 5. scene_info (场景信息) 字段详解

### 5.1 m_info (节点自身信息)

#### 5.1.1 speed (速度)
- **类型**: `double` (m/s)
- **来源**: `ipsRouting->GetSceneInfo().m_info.speed`
- **底层数据源**: 节点的 MobilityModel
  ```cpp
  node->GetObject<MobilityModel>()->GetVelocity().GetLength()
  ```

#### 5.1.2 energy (剩余能量)
- **类型**: `double`
- **来源**: `ipsRouting->GetSceneInfo().m_info.energy`
- **底层数据源**: 能量模型（EnergySourceContainer）
  ```cpp
  node->GetObject<EnergySourceContainer>()->Get(0)->GetRemainingEnergy()
  ```

#### 5.1.3 queue_length (队列长度)
- **类型**: `uint32_t`
- **来源**: `ipsRouting->GetSceneInfo().m_info.queue_length`
- **底层数据源**: 请求队列
  ```cpp
  m_queue.GetSize()  // RequestQueue 的当前大小
  ```

#### 5.1.4 neighbor_count (邻居数量)
- **类型**: `uint32_t`
- **来源**: `ipsRouting->GetSceneInfo().m_info.neighbor_count`
- **底层数据源**: 位置表
  ```cpp
  m_neighbors.size()  // PositionTable 中的邻居条目数
  ```

#### 5.1.5 distance_to_destination (到目标的距离)
- **类型**: `double` (米)
- **来源**: `ipsRouting->GetSceneInfo().m_info.distance_to_destination`
- **底层数据源**: 计算当前位置与目标位置的欧式距离
  ```cpp
  CalculateDistance(currentPosition, destinationPosition)
  ```

### 5.2 neighbor_info (邻居统计信息)

以下所有字段均通过遍历 PositionTable 中的所有邻居节点，统计计算得出：

#### 5.2.1 forward_candidate_ratio (前向候选比例)
- **类型**: `double` (0.0 - 1.0)
- **计算方式**: 距离目标更近的邻居数 / 总邻居数
- **含义**: 可作为下一跳转发的候选邻居占比

#### 5.2.2 distance_to_me_mean / std (到本节点距离的均值/标准差)
- **类型**: `double` (米)
- **计算方式**: 
  - 对所有邻居计算: `CalculateDistance(myPosition, neighborPosition)`
  - 求均值和标准差

#### 5.2.3 distance_to_destination_mean / std / min
- **类型**: `double` (米)
- **含义**: 所有邻居到目标节点距离的统计值
- **计算方式**: 遍历邻居，计算每个邻居到 destination 的距离，统计均值/标准差/最小值

#### 5.2.4 relative_speed_mean / std (相对速度)
- **类型**: `double` (m/s)
- **计算方式**:
  ```cpp
  Vector myVelocity = myNode->GetObject<MobilityModel>()->GetVelocity();
  Vector neighborVelocity = neighborNode->GetObject<MobilityModel>()->GetVelocity();
  double relativeSpeed = (myVelocity - neighborVelocity).GetLength();
  ```
- **含义**: 本节点与邻居的相对运动速度统计

#### 5.2.5 link_lifetime_mean / std (链路生存时间)
- **类型**: `double` (秒)
- **计算方式**: 根据相对速度和通信范围估算链路可维持时间
  ```cpp
  // 简化公式: lifetime ≈ (通信范围 - 当前距离) / 相对速度
  ```

#### 5.2.6 neighbor_degree_mean / std (邻居度数)
- **类型**: `double`
- **含义**: 邻居的邻居数量统计（二跳信息）
- **数据源**: 从邻居的 HELLO 消息中获取其邻居表大小

#### 5.2.7 queue_length_mean / std / max (邻居队列长度)
- **类型**: `double` / `uint32_t`
- **含义**: 邻居节点的数据包队列长度统计
- **数据源**: 从邻居通告的信息中获取

#### 5.2.8 energy_mean / std / min (邻居能量)
- **类型**: `double`
- **含义**: 邻居节点的剩余能量统计
- **数据源**: 从邻居 HELLO 消息携带的能量信息获取

## 6. para_info (参数信息) 字段详解

### 6.1 hello_interval (Hello 消息间隔)
- **类型**: `double` (秒)
- **来源**: `ipsRouting->GetParaInfo().hello_interval`
- **说明**: 协议的 HELLO 消息发送间隔，用于维护邻居表

### 6.2 path_num (路径数量)
- **类型**: `uint32_t`
- **来源**: `ipsRouting->GetParaInfo().path_num`
- **说明**: 多路径路由的路径条数配置

### 6.3 weights (权重参数)

#### 6.3.1 w_distance (距离权重)
- **类型**: `double`
- **来源**: `ipsRouting->GetParaInfo().weights.w_distance`
- **说明**: 路由决策中距离因子的权重

#### 6.3.2 w_linkTime (链路时间权重)
- **类型**: `double`
- **来源**: `ipsRouting->GetParaInfo().weights.w_linkTime`
- **说明**: 路由决策中链路生存时间因子的权重

#### 6.3.3 w_relVelocity (相对速度权重)
- **类型**: `double`
- **来源**: `ipsRouting->GetParaInfo().weights.w_relVelocity`
- **说明**: 路由决策中相对速度因子的权重

#### 6.3.4 w_neighborCount (邻居数权重)
- **类型**: `double`
- **来源**: `ipsRouting->GetParaInfo().weights.w_neighborCount`
- **说明**: 路由决策中邻居数量因子的权重

**权重使用场景**: 在选择下一跳节点时，综合评分公式类似：
```
Score = w_distance * distance_metric 
      + w_linkTime * linktime_metric 
      + w_relVelocity * velocity_metric 
      + w_neighborCount * neighbor_metric
```

## 7. result_info (结果信息) 字段详解

> **重要说明**: result_info 中的所有统计数据均为**周期性平均值**，即每个上报周期（例如 10 秒）内的统计结果。每次上报后，通过 `ipsRouting->ResetForUpload()` 重置计数器，下一个周期重新开始统计。

### 7.1 avg_pdr (平均分组投递率)
- **类型**: `double` (0.0 - 1.0)
- **来源**: `ipsRouting->GetResultInfo().avg_pdr`
- **统计周期**: 上次上报到本次上报之间的时间窗口（例如 10 秒）
- **计算方式**: 
  ```cpp
  PDR = 本周期内成功接收的数据包数 / 本周期内发送的数据包总数
  ```
- **数据源**: 协议内部维护的发送/接收计数器（每次上报后清零）

### 7.2 avg_delay (平均端到端延迟)
- **类型**: `double` (秒)
- **来源**: `ipsRouting->GetResultInfo().avg_delay`
- **统计周期**: 上报周期内（例如 10 秒）
- **计算方式**:
  ```cpp
  avg_delay = Σ(接收时间 - 发送时间) / 本周期内成功接收包数
  ```
- **数据源**: 数据包头部携带的时间戳，仅统计本周期内接收的数据包

### 7.3 energy_consumption (能量消耗)
- **类型**: `double` (焦耳)
- **来源**: `ipsRouting->GetResultInfo().energy_consumption`
- **统计周期**: 上报周期内（例如 10 秒）
- **计算方式**:
  ```cpp
  energy_consumption = 上次上报时的剩余能量 - 本次上报时的剩余能量
  ```
- **说明**: 仅统计本周期内的能量消耗增量

### 7.4 control_packets (控制包数量)
- **类型**: `uint32_t`
- **来源**: `ipsRouting->GetResultInfo().control_packets`
- **统计周期**: 上报周期内（例如 10 秒）
- **统计内容**: 
  - HELLO 消息发送数量
  - RERR (路由错误) 消息数量
  - 其他协议控制消息数量
- **数据源**: 协议内部计数器，每发送一个控制包 +1，每次上报后清零

### 7.5 distance_progress (距离推进)
- **类型**: `double` (米)
- **来源**: `ipsRouting->GetResultInfo().distance_progress`
- **统计周期**: 上报周期内（例如 10 秒）
- **计算方式**: 
  ```cpp
  distance_progress = 上次上报时到目标的距离 - 本次上报时到目标的距离
  ```
- **重置机制**: 每次上报后调用 `ipsRouting->ResetForUpload()` 将累计量清零
- **含义**: 正值表示向目标靠近，负值表示远离目标

## 8. 数据获取调用链总结

```
UploadLoop_Comm
  └─ BuildNodeReport(node, nodeId)
       ├─ node->GetObject<Ipv4>()->GetRoutingProtocol()
       ├─ DynamicCast<intelligentprotocolstack::RoutingProtocol>(routingProtocol)
       └─ ipsRouting 提供的三个 getter 方法:
            ├─ GetSceneInfo()   → 返回 SceneInfo 结构体
            │   ├─ m_info: 从 MobilityModel, EnergySource, Queue 实时读取
            │   └─ neighbor_info: 遍历 PositionTable 统计计算
            ├─ GetParaInfo()    → 返回 ParaInfo 结构体
            │   └─ 读取协议配置的静态参数
            └─ GetResultInfo()  → 返回 ResultInfo 结构体
                └─ 读取协议维护的性能统计计数器
```

## 10. 注意事项

1. **节点过滤**: 仅上传 `node_identity != 0` 的节点，身份为 0 的节点被视为非活跃节点
2. **单节点上传**: 每次 HTTP 请求仅包含一个节点的数据（`nodes` 数组只有 1 个元素）
3. **同步通信**: 发送后立即阻塞等待 Windows 端响应，然后再发送下一个节点
4. **连接管理**: 每次上传前重新建立 TCP 连接，请求结束后关闭连接（`Connection: close`）
5. **数据重置**: 每次上传后调用 `ResetForUpload()` 重置累计型指标（如 distance_progress）
6. **精度控制**: 浮点数保留 2 位小数（`std::setprecision(2)`）

## 11. 扩展说明

### 11.1 非 IPS 协议节点的处理
如果节点使用的不是 Intelligent Protocol Stack (IPS) 协议：
- `IsActiveNode()` 默认返回 `true`（保持向后兼容）
- `BuildNodeReport()` 中所有结构体字段填充空值/零值
- 仍会进行上传，但数据无实际意义

### 11.2 时间调度机制
使用 NS-3 的事件调度器 `Simulator::Schedule`:
```cpp
Simulator::Schedule(Seconds(rp->m_uploadInterval), &UploadLoop_Comm, rp);
```
- 在仿真时间（非真实时间）上调度下一次上传
- `m_uploadInterval` 控制上传周期（秒）
